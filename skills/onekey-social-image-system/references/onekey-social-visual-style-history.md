# OneKey 社媒视觉设计风格规范

本规范基于 OneKey 历史社媒案例归纳，用于指导 OneKey 社媒图的版式、视觉风格、图片选择和最终 QA。它描述的是历史案例形成的视觉语言；实际生产时仍需同时遵守当前品牌 token、字体规范、logo 使用规范和具体活动 brief。

## 1. 整体视觉定位

OneKey 社媒视觉属于 `Modern Tech + FinTech Editorial Design`。

品牌关键词：

- Web3
- 金融科技 FinTech
- 专业可信
- 极简现代
- 数据化
- 国际化

核心风格特征：

- 信息效率优先
- 强模块化设计
- 极简布局
- 大量留白
- 品牌色高识别
- 视觉元素统一复用
- 兼具科技产品视觉、金融数据可视化和品牌营销传播

## 2. 色彩体系

### OneKey Green

OneKey Green 是社媒视觉中最强的品牌识别资产。实际生产必须以用户提供的颜色色值规范和当前 OneKey social brand token 为准，不从历史案例中推断或扩展绿色区间。

使用场景：

- CTA
- 描边
- 数据高亮
- 功能标签
- 信息区块
- 主要品牌背景或视觉锚点

典型案例类型：

- Wallet Integration
- New Chain Supported
- Top Swaps
- Builder Fee 系列

执行注意：

- 社媒图中的布局绿应以当前 OneKey social brand token 为准。
- 官方 logo 内部的绿色不可手动改色。
- 绿色 logo / icon 的使用必须遵守 `onekey-logo-usage.md`：绿色 logo / icon 仅用于黑色背景；OneKey Green 和白色背景均使用黑色 logo / icon。

### 黑色

黑色用于建立科技感、金融感和高对比视觉。

常见用途：

- 科技感背景
- 数据展示
- 产品展示
- 深色活动或交易类视觉

历史案例中黑色占比约 40%，常见于：

- Perps
- Referral
- 年终活动
- Trophy 活动

### 白色

白色用于承载信息、提升阅读效率和建立专业感。

常见用途：

- 信息卡片
- 教程内容
- 功能公告
- 产品说明

历史案例中白色占比约 40%，常见于：

- Builder Fee 系列
- New Chain Supported
- Wallet Integration

### 活动色

特殊运营活动允许在品牌色之外加入活动色，但必须由主题或活动语境驱动。

- 红色：春节、节日活动、限时营销
- 紫色：Trading Carnival、Perps 活动
- 金色：奖杯、成就、荣誉感内容

活动色规则：

- 活动色不能替代 OneKey Green 的品牌识别角色。
- 活动色应作为主题层或重点视觉，不应导致品牌识别丢失。
- 仍需保留黑、白、OneKey Green 中至少一个强品牌锚点。

## 3. 版式体系

### 卡片化设计

历史社媒图大量采用 Card Layout，适合在社媒流中独立阅读和批量生产。

典型结构：

```text
┌─────────────┐
│ Logo        │
│             │
│ 主视觉      │
│             │
│ 标题        │
│ 描述文案    │
└─────────────┘
```

特点：

- 信息边界清晰
- 适合社媒信息流快速浏览
- 易于扩展为系列化模板
- 有利于统一 logo、标题、产品截图和 CTA 的相对位置

### 栅格系统

单栏布局占比最高，适用于：

- 功能公告
- 新资产支持
- 教程
- 单一核心卖点

双栏布局适用于：

- 前后对比
- Step 教程
- 数据展示
- 两组信息的并列解释

执行原则：

- 优先选择单栏，让读者 1 到 2 秒内理解主题。
- 双栏只在信息确实需要并列比较时使用。
- 不为了装饰而拆分网格。

## 4. 排版规范

### 标题层级

一级标题：

- 超大字号
- Bold
- 强视觉冲击
- 通常承载唯一主信息

二级标题：

- SemiBold
- 控制在 1 到 2 行
- 用于副标题、功能名、活动说明

三级说明：

- 小字号
- 灰度弱化
- 用于时间、产品说明、补充信息、注释

### 排版原则

左对齐优先：

- 信息读取效率最高
- 符合国际化阅读习惯
- 适合金融科技类信息密度

大留白：

- 白底内容和功能介绍尤其需要大量留白
- 留白用于建立高级感、科技感和可信度
- 不要用装饰元素填满空白

控制文案长度：

- 高频使用单行标题，例如 `Wallet Integration`
- 允许两行标题，例如 `Bluetooth / Connection Support`
- 避免把完整说明文案塞进视觉主标题

## 5. 视觉元素体系

### 产品截图

产品截图是 OneKey 社媒中出现频率最高的视觉元素。

使用场景：

- App 功能
- 新版本
- 钱包界面
- 操作路径
- 产品能力展示

规范：

- 使用真实 UI
- 手机 Mockup 或产品容器角度要统一
- 截图要清晰、可读、可检查
- 产品截图服务于信息，不作为无意义装饰

### Token / Coin Icon

Token 和 Coin Icon 大量用于：

- New Asset Supported
- Market
- Perps
- 新链或新资产支持

规范：

- 使用来源明确的官方图标或用户提供资产
- 不允许 AI 重绘或臆造
- 图标应居中、清晰、有充足留白

### 线框与描边

品牌绿描边是常用视觉语言。

用途：

- 信息分区
- 强调模块
- 卡片边框
- 标签和 CTA
- 数据高亮

规范：

- 描边应重复使用同一粗细和圆角逻辑
- 不要混用太多描边样式
- 线框服务于结构，不做随机装饰

## 6. 图片使用规范

### 产品展示

产品展示在历史案例中占比约 60%。

特点：

- 产品即主角
- 背景弱化
- 信息与产品强绑定

执行：

- 优先使用真实产品、真实 UI、真实界面。
- 如果产品图或 UI 不清楚，应先索取或制作干净素材，而不是用抽象图替代。

### 人物素材

人物素材占比极低，仅在以下场景出现：

- AMA
- KOL 合作
- 活动嘉宾
- 社区人物内容

原则：

- 人物服务于内容，不是视觉主体。
- 人物不能削弱 OneKey 品牌识别。
- 人物图应干净、专业、可信，避免过度生活方式化。

### 3D 物料

3D 物料风格偏真实渲染，非卡通。

规范：

- 可用于硬件产品、活动物料、奖杯、成就感内容。
- 避免卡通化、玩具化、过度夸张。
- 3D 元素必须服务于产品或活动主题。

## 7. 出图执行原则

每次出图前，先判断内容类型：

- 产品功能：优先产品截图 + 单栏信息结构
- 新资产/市场：优先 Token/Coin Icon + 数据化标题
- 教程：优先白底卡片 + 分步信息
- 活动营销：允许活动色，但保留品牌锚点
- 荣誉/成就：可加入金色或奖杯视觉，但保持极简

默认优先级：

1. 信息清晰
2. 品牌识别
3. 产品真实
4. 模块统一
5. 视觉冲击

不要为了视觉冲击牺牲信息效率。

## 8. QA Checklist

出图或审图时检查：

- 是否一眼能识别为 OneKey？
- 是否符合 Web3 / FinTech / 专业可信 / 极简现代 / 数据化 / 国际化？
- 是否使用了清晰模块，而不是散乱元素？
- 是否有足够留白？
- 标题是否足够短、足够大、足够明确？
- 是否优先左对齐，或有明确理由使用居中 / 非对称布局？
- 绿色是否作为品牌识别或关键高亮，而不是随机铺色？
- 活动色是否有明确活动理由？
- 产品截图、Token 图标、3D 物料是否真实、清晰、来源明确？
- 人物素材是否仅在 AMA / KOL / 嘉宾等必要场景使用？
- logo 是否同时通过 `onekey-logo-usage.md` 的规则检查？
- C.R.A.P. 是否通过：Contrast、Repetition、Alignment、Proximity？
